**Usenet Ultimate v1.3.0**

	Hey everyone\! I'm back with the latest update for Usenet Ultimate. It's a big one.  
	  
	For those that don’t know, Usenet Ultimate is a self-hosted Stremio add-on for streaming Usenet content. It searches indexers (Newznab/Prowlarr/NZBHydra) and EasyNews, sends NZBs to NZBDav for virtual mounting, and streams the resulting video files back through Stremio (or other Stremio add-on supported apps) via a WebDAV proxy or direct redirect to the WebDAV mounted address.

Usenet Ultimate is the original home of features like NZB Fallback, Proxy-less WebDAV Streaming Mode, Dedicated Season Pack Support, Smart Mode Health Checking, Auto Indexer Capability Discovery, NZB Databases, Reboot/Remake Detection, NZB Fallback Auto-Resolve, and more\!

This update is the product of 5 public beta tests, a release candidate, an immense amount of community feedback, feature requests, bug reports, and my own original ideas. I’d like to give a special and enthusiastic, THANK YOU, to all of the Usenet Ultimate community members. You've helped to make updates like this a reality.

1) **General**

- Multi-Manifest support   
  - Usenet Ultimate now supports the creation of additional manifests per account with customizable names. Up to 25 total.   
  - All manifests can be renamed, regenerated, and deleted.  
  - The new menu and options for this are located in the “Install” tab.

    

- Trust Proxy **(Credit: boopsielol \- PR \#23)**  
  - When running behind a reverse proxy that terminates TLS (nginx, Caddy, Traefik), Express was ignoring the proxy's X-Forwarded-Proto header and always seeing the internal http connection. This caused stream redirects and fallback URLs to be constructed as http instead of https, breaking clients like Infuse that don't follow the protocol upgrade.   
  - With trust proxy now enabled, Express correctly reads the forwarded protocol and returns https URLs. No effect when not behind a reverse proxy.

    

- Dynamic Base URL  
  - Stream URLs are now automatically derived from the incoming request's protocol and host. Clients connecting via LAN IP, custom domain, or localhost all receive reachable stream URLs without configuration.  
  - This means the BASE\_URL environment variable is optional while also still supporting the environment variable as an override for custom domains or Docker networking.

    

- Stremio Path Prefix  
  - Addon routes are now available at both /stremio/{uuid}/ and /{uuid}/ paths. New installations default to the /stremio/ prefix, which simplifies reverse proxy setups (e.g., Traefik can route /stremio/\* publicly while keeping the rest behind auth).   
  - Existing installations continue to work at the root-level path without any changes.   
  - Stream URLs, video proxy redirects, and fallback chains all respect the path used by the client.

- External Player Filenames  
- External video players now display the “Clean Title” ("MovieTitle (Year)" or "ShowTitle S\#\#E\#\#") instead of a raw URL or generic “stream” name.  
  - This is the only solution because most external players use the video URL path as the filename, which cannot be dynamic while supporting NZB Fallback.

	**Remake / Reboot Protection:**  
	  
Two layer reboot/remake detection: 

- This feature prevents results from the wrong version of a show (original vs remake) from appearing in Text and ID search results. It’s constructed of two independent layers working together.  
    
  - Layer 1: Year check (always active, applies to all shows)  
    - If a release contains a year that differs from the expected year by \>1 \= rejected  
    - Handles years anywhere in the title (before/after episode markers, after episode name text) by using virens parsing library (Discussed more in **10\) Filters & Sorting**)  
        
  - Layer 2: Episode Name Check (remake-flagged shows and year-less releases only)  
    - I use TVDB to detect if a show has a known remake by finding 2+ series with the same title but different years.  
    - For year-less releases on remake-flagged shows the release must contain the expected TVDB episode name or it's rejected.  
    - Enabled by default and toggleable in the UI.

- Reboot/Remake detection is a global feature that filters all results, both ultimate text search and id searches

|  | Text search | ID-based (non-remake) | ID-based (remake) |
| ----- | ----- | ----- | ----- |
| Year \+ matches (±1) | ACCEPT | ACCEPT (no check) | ACCEPT |
| Year \+ doesn't match | REJECT | ACCEPT (no check) | REJECT |
| No year \+ no remake | ACCEPT | ACCEPT | n/a |
| No year \+ ep name \+ remake | ACCEPT | n/a | ACCEPT |
| No year \+ no ep name \+ remake | REJECT | n/a | REJECT  |


  


  


  


  


- Season packs are included in remake/reboot detection:  
  - **Season Pack Decision Tree**:      **Has Remake**  
  - Year \+ matches (±1)	                    KEEP  
  - Year \+ doesn't match	                    REMOVE  
  - No year		                    DEPRIORITIZE

    

- No year season packs are deprioritized because it cannot be determined what the actual content is of. To preserve as many results as possible especially for older shows, these season packs are deprioritized to the end of the results list.

5) **NZB Fallback**

- Bug Fix: Prevent Stremio timeouts on the initial stream request if it is \> 60s  
  - The initial candidate is placed within the same redirect chain as all other candidates preventing a 60s timeout from ever occuring..

    

- NZB wait times are now capped at 90 seconds instead of 3 minutes.


- Bug Fix: Falling back to a season pack from an individual episode wasn't passing the episode number  
  - Usenet Ultimate now uses the episode number that’s stored in the fallback group to ensure the correct episode is selected.

    

- Season packs now support their own NZB Fallback wait time  
  - The default wait time is 30 seconds, this allows for additional nzbdav processing time over a single episode file, as season packs can be much larger nzbs.  
  - A season pack identifier is set per candidate which allows for switching between multiple wait times within a single fallback group when alternating between single episode files and season packs.  
    - This means NZB Fallback is content aware, knowing which wait time to use for which type of result (episode vs season pack) on processing.

      

- Proxy Streaming Method: Fallback on Server Errors:  
  - When WebDav returns a server error (500) for a video file, the proxy now triggers NZB Fallback to the next available candidate in the results list instead of forwarding the error response to the player.

    

  - Previously, non-2xx (except for 404s) responses were passed directly to the player, causing it to crash with "No valid length mask found" when trying to parse error data as video.

    

- Broken Video Path Tracking Awareness  
  - Video paths that return server errors from WebDav are temporarily marked as broken.   
  - This prevents the library check from repeatedly returning a file that exists on disk but can't actually be served, and prevents wasted NZB re-submissions that resolve to the same unservable file, and which result in the same server error (500).   
  - Broken paths share the same TTL as fallback groups.

    

- Differentiated Error Handling  
  - 404/410 errors permanently mark the NZB as dead so it won't be retried.   
  - 5xx errors do not blacklist the NZB into the Dead NZB Database, allowing for it to be retried in future searches, once the server recovers.

    

- Security: Block Open Redirect in Fallback URL  
  - The fallback redirect parameter is now validated to reject protocol-relative URLs, preventing potential open redirect attacks through crafted stream URLs.

    

- **Auto-Resolve on Search:**


  - When "From Top of List" fallback is active and “Auto-Resolve on Search” is active within it, NZBs are automatically resolved in the background as soon as search results are returned. Candidates are tried sequentially from the top of the results list until the first healthy result is found.   
  - In this case when you click a stream, the results have already started processing in the background, which, depending on the results, means instant playback with no wait time at best, and a faster resolved fallback chain at worst. Decreasing the time it takes to start streaming across the board.  
  - This also works with auto play.   
    - When Stremio scrapes the next episode in the background, auto-resolve kicks in immediately. By the time your current episode ends, the next one has already been Auto-Resolved and ready to play.  
  - Logging has been added so you can watch the Auto-Resolve process in real-time.  
  - Some examples:  
    - If auto-resolve has already tried candidates \#1 and \#2 (both failed and entered into the dead nzb db) and successfully resolved \#3, then you click candidate \#5 which in this case fails, the fallback loop sees the dead nzb db entries for \#1 and \#2 instantly iterates past them, and lands on \#3 for instant playback.  
    - If you click a stream while Auto-Resolve is still in-progress, your request merges with the existing in-progress Auto-Resolve process rather than starting a duplicate.

      

  - Auto-Resolve relies on the WebDAV library checks  
    - Library checks are forced on when Auto-Resolve is enabled.  
    - When Auto-Resolve is turned off, the library check is returned to its previous state.

      

  - NZB Fallback Default Changes:  
    - Because this is now a flagship feature of NZB Fallaback, its defaults have changed. Existing users updating from 1.2.0 will not have their configs changed.  
    - "From Top of List" is now the default fallback order.   
      - Your selected stream is still tried first, then fallback proceeds from the top of the results list.   
        - The fallback order description has been updated to reflect this: "Try the stream you selected first, then fall back from the top of the results list."  
      - Existing user configs are not changed.  
    - “Auto Resolve on Search” is enabled by default  
      - Existing user configs are not changed.

        

  - The NZB Fallback card now shows "+ Auto-Resolve" in amber when auto-resolve is active.

11) **Filters & Sorting**

- **Mandatory Filter and Sorts \- Reset to Defaults:**  
- Because the entire filter and sort menu has been rewritten there is a one-time auto reset to everyone's filter and config options.  
- **Usenet Ultimate now uses virens (AIOStreams) fork of the parse-torrent-title package** (shout out to him for his fork, the github here: [https://github.com/Viren070/parse-torrent-title](https://github.com/Viren070/parse-torrent-title)) 

- On-top of the use of the new parser, I have made additional normalizations and customizations which are used both alongside the parsing package and in some instances in place of it. 


- **Resolution**  
  - Uses parsed.resolution from library.   
  - Custom Logic:  
    - Usenet Ultimate normalizes '2160p' and '4k' both to '4k'.   
    - Falls back to raw title detection for 'UHD' and 'UHDRip' when the library returns no resolution, mapping both to '4k'. (The library does not support this)  
    - 540p catch logic  
    - Added 540p to the resolution filter and priority list.  
    - All other values pass through lowercase as the library returns them ('1080p', '720p', '576p', '480p', etc.).  
  - Supported Values: 4k, 1440p, 1080p, 720p, Unknown, 576p, 540p, 480p, 360p, 240p, 144p

- **Codec**  
  - Uses parsed.codec from library  
    - Usenet Ultimate then groups the equivalent codecs:  
- hevc, h265, x265 → hevc  
- x264, h264, avc → avc  
- xvid, divx, dvix → xvid  
- Everything else passes through as-is (av1, vp9, vp8, mpeg2)  
- Supported values: av1 → hevc → vp9 → avc → vp8 → xvid → mpeg2 → Unknown

- **Quality**  
- Uses parsed.quality from library  
- Supported values: BluRay REMUX → REMUX → BDMUX → BRMUX → BluRay → WEB-DL → WEB → DLMUX → UHDRip → BDRip → WEB-DLRip → WEBRip → BRRip → WEBCap → VODR → HDTV → HDTVRip → SATRip → TVRip → PPVRip → DVD → DVDRip → PDTV → SDTV → HDRip → SCR → WORKPRINT → TeleCine → TeleSync → CAM → VHSRip → Unknown.

- **HDR**  
- Uses parsed.hdr, parsed.bitDepth, and parsed.upscaled from library  
- Custom logic is:  
  - If array has DV \+ any other HDR value → 'HDR+DV' (combo tag)  
  - If DV alone → 'DV'  
  - Otherwise → first element returned   
  - If no hdr array returned: check parsed.bitDepth \=== '10bit' → '10bit', parsed.upscaled \=== true → 'AI'  
  - More Custom Logic: “3D”   
  - It defaults to disabled in the priority list, filtering out 3D results  
    - The parser library isn’t able to detect all types of 3D. TV Shows are unsupported by the library.  
    - I’ve added custom logic to support 3d filtering for tv shows  
    - If 3d exists after S\#\#E\#\# it is 3d  
- Supported Values: DV, HDR+DV, HDR10+, HDR, 10bit, AI, SDR, 3D (disabled by default), Unknown

- **Audio**  
- Uses parsed.audio from library  
- Custom logic is:  
  - \['Atmos', 'TrueHD'\] → Atmos (TrueHD)  
  - \['Atmos', 'DDP'\] or \['Atmos', 'EAC3'\] → Atmos (DDP)  
  - \['Atmos'\] returned by itself, infers from source:   
    - Blu-ray sources (BluRay, REMUX, BDRip, BRRip, UHDRip, BDMUX, BRMUX) → Atmos (TrueHD)  
    - everything else → Atmos (DDP)  
  - EAC3 → DDP  
  - AC3 → DD  
  - No Atmos → first element as-is

- Supported values: Atmos (TrueHD), DTS Lossless, TrueHD, Atmos (DDP), DTS Lossy, DDP, DD, FLAC, PCM, AAC, OPUS, MP3, Unknown


- Notes: DTS-HD MA and DTS:X are both 'DTS Lossless' (library can't distinguish). DTS and DTS-HD HR are both 'DTS Lossy'.   
- EAC3 is normalized to DDP (same codec: Dolby Digital Plus).   
- AC3 is normalized to DD (same codec: Dolby Digital).

- **Language**  
- No changes have been made to language. It was already using the parse-torrent-title library, parsed.languages and parsed.dubbed results, with the following custom logic:  
- Custom logic:  
  - No languages detected \+ dubbed flag → 'Dubbed'  
  - No languages detected \+ no dubbed flag → 'English' (assumes English when no tag, following official scene release rules)  
  - 'multi audio' or 'multi subs' in array → 'Multi'  
  - 'dual audio' in array → 'Dual Audio'  
  - More than 1 language → 'Multi'  
  - Single language → look up ISO 639 code in LANG\_CODE\_TO\_DISPLAY map (e.g., 'fr' → 'French', 'es-419' → 'Latino')  
  - Code not in map → 'Unknown'  
- Notes: There’s no normalization of the library values themselves, the library returns language codes ('en', 'fr', 'ja', etc.) and special strings ('multi audio', 'dual audio', 'multi subs'). Usenet Ultimate just translates codes to readable names.  
- Supported Values: English, Multi, Dual Audio, Dubbed, Arabic, Bengali, Bulgarian, Chinese, Croatian, Czech, Danish, Dutch, Estonian, Finnish, French, German, Greek, Gujarati, Hebrew, Hindi, Hungarian, Indonesian, Italian, Japanese, Kannada, Korean, Latino, Latvian, Lithuanian, Malay, Malayalam, Marathi, Norwegian, Persian, Polish, Portuguese, Punjabi, Romanian, Russian, Serbian, Slovak, Slovenian, Spanish, Swedish, Tamil, Telugu, Thai, Turkish, Ukrainian, Vietnamese

- **Edition**  
- Uses parsed.edition, parsed.unrated, and parsed.uncensored from library.   
- If parsed.edition exists → return it directly.  
- If parsed.edition doesn't exist:  
  - If parsed.unrated is true \= Unrated  
  - If parsed.uncensored is true \= Uncensored  
  - If parsed.edition, parsed.unrated, and parsed.uncensored are all empty or false, use custom logic.  
  - If no matches at all, mark as standard.  
- Custom logic for editions that are unsupported by the library:  
  - Title check for “Superfan”  
  - Title check for abbreviated Directors Cut ( .dc. or \-dc- or ends with .dc)  
  - Title check for Special Edition  
- Extended editions are displayed as “Extended”.  
  - The custom logic for Prefer Non-Standard Editions (Prioritize all enabled non-standard editions equally over Standard) continues to exist. Disabled by default.  
- Supported Values: Extended Edition, Director's Cut, Superfan, Unrated, Uncensored, Uncut, Theatrical, IMAX, Special Edition, Collector's Edition, Criterion Collection, Ultimate Edition, Anniversary Edition, Diamond Edition, Dragon Box, Color Corrected, Remastered, Standard

- **Release Group**  
- Uses parsed.group from library  
- If no result is returned it’s set to unknown

- **Post Age**   
  - Post Age is a new sorting method that is based on the age of each NZB (e.g., 2h, 3d, 1.2y) based on its publish date.  
  - EasyNews:   
    - extracts posted timestamp from API response (item\[8\])  
  - Newznab/NZBHydra:   
    - Uses the pubDate from the XML response  
  - Prowlarr:  
    - Uses the publishDate from the API response  
  - Has configurable stream viewer element with 📅 emoji, enabled by default

- **Bitrate**   
  - Bitrate is a new sorting method that displays the estimated video bitrate (e.g., 8.5 Mbps, 5.2 Mbps)  
  - EasyNews:   
    - Uses exact bitrate from actual file duration (item\[14\])  
  - Newznab/Prowlarr/NZBHydra:   
    - When a runtime isn’t directly provided from an indexer (most cases), Usenet Ultimate uses the estimated bitrate calculated with a runtime from TMDB for movies,  TVDB for TV, and Cinemeta as a fallback.  
  - Season packs use the per-episode estimated size for bitrate calculation, not full pack size.  
  - Configurable stream viewer element with 📊 emoji, disabled by default.
